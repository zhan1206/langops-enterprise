## 一、行业背景与核心痛点深度拆解

### 行业背景

2026年企业级AI应用已从demo验证期全面进入规模化生产落地期，90%以上的企业AI应用并非基于自研/微调大模型，而是基于商用大模型API，通过Prompt工程、RAG检索增强、Agent编排实现业务闭环。这类应用的开发、测试、发布、运维逻辑，与传统软件、传统模型训练完全不同，现有工具体系完全无法适配。

当前开源生态的核心空白：闭源商业工具LangSmith几乎垄断了LLM应用Ops市场，但价格高昂、强绑定LangChain生态、不支持私有化部署；开源生态中无一套可支撑生产环境落地的、全生态兼容的、覆盖LLM应用全生命周期的一站式Ops平台。企业要么被迫付费使用闭源工具，要么只能零散拼凑Prompt管理、RAG评测、日志监控等单功能工具，研发与运维成本极高。

### 核心痛点与解决方案对照表

## 二、项目整体架构设计

项目采用云原生分布式架构，高内聚低耦合，全模块可扩展、可插拔，能力原生内置而非插件扩展，从上到下分为5层核心架构：

```
┌─────────────────────────────────────────────────────────┐
│ 前端交互与协作管控层 │
│ 可视化编排、版本管理、协作评审、模板市场、管控控制台 │
├─────────────────────────────────────────────────────────┤
│ 安全合规与全链路审计层 │
│ 权限管控、不可篡改审计、内容安全、敏感数据防护 │
├─────────────────────────────────────────────────────────┤
│ 全生命周期发布管控层 │
│ 环境隔离、灰度发布、A/B测试、自动回滚、流量调度 │
├─────────────────────────────────────────────────────────┤
│ 评测与持续优化核心层（项目心脏） │
│ 自动化评测、回归测试、效果退化检测、根因分析与优化 │
├─────────────────────────────────────────────────────────┤
│ 全生态兼容与可观测层 │
│ 多框架适配器、模型适配、全链路追踪、监控告警、成本管控 │
└─────────────────────────────────────────────────────────┘
```

### 1. 前端交互与协作管控层

项目的用户交互入口，解决团队协作混乱、开发效率低的痛点：

- 低代码可视化编排平台：提供可视化画布，零代码拖拽即可搭建Prompt流程、RAG检索链路、Agent工作流
- 全链路版本管理系统：支持Prompt、RAG配置、Agent工作流、测试用例的全版本管理
- 团队协作与评审管控中心：内置企业级团队协作机制，支持用户/团队/角色分级权限管理
- 开箱即用模板市场：内置客服问答、代码辅助、知识库问答等20+场景标准化模板
- 统一管控控制台：企业管理员一站式全局管控中心

### 2. 安全合规与全链路审计层

企业级落地的红线能力：

- 细粒度RBAC权限管控模型：支持用户/团队/项目/角色4级分级权限管理
- 全链路不可篡改审计系统：全程留痕，链式存储+加密签名，不可篡改
- 原生内容安全护栏：多维度内容安全检测引擎
- 敏感数据全生命周期防护体系：内置100+敏感数据识别规则
- 合规规则库：内置等保2.0、GDPR、SOX等主流合规规则

### 3. 全生命周期发布管控层

项目的核心管控能力：

- 多环境严格隔离体系：支持开发/测试/预发/生产4级环境隔离
- 多模式灰度发布引擎：支持多维度灰度放量
- 全场景A/B测试平台：支持多版本并行测试
- 多���件自动回滚机制：支持多维度阈值设置
- 智能流量调度引擎：可自定义流量分发规则

### 4. 评测与持续优化核心层（项目核心心脏）

项目的差异化核心壁垒：

- 多维度自动化效果评测引擎：支持10+维度的自动化量化评测
- 自动化回归测试体系：支持批量自动化测试、版本对比测试
- 实时效果退化检测引擎：7*24小时监控线上应用效果
- 智能根因分析与优化引擎：自动定位问题根因，生成优化建议
- 标注与反馈闭环体系：支持用户反馈、人工标注

### 5. 全生态兼容与可观测层

项目的基础底座：

- 多框架全兼容适配器：为LangChain、LlamaIndex、AutoGen等开发标准化适配插件
- 全模型与工具适配能力：兼容所有主流商用/开源大模型
- 全链路可观测平台：基于OpenTelemetry开源标准
- 自定义监控与告警体系：内置核心指标监控大盘
- 多维度成本管控与优化引擎：支持精细化成本统计

### 场景1：企业内部知识库问答系统全生命周期管控

落地方式：企业内部知识库RAG问答系统全量接入LangOps，实现Prompt、RAG检索配置的全版本管理、团队协同开发；通过自动化评测体系测试问答的准确率、事实一致性、合规性；上线后通过灰度发布平滑迭代，实时监控效果。

核心价值：将知识库问答系统的准确率从70%提升至95%以上，迭代周期从1周缩短至1天。

## 八、快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/zhan1206/langops-enterprise.git
cd langops-enterprise

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp .env.example .env
# 编辑 .env 填入你的API Key

# 启动服务
python run.py server
```

### 快速开始

```bash
# 创建第一个应用
python run.py app create --name "我的问答助手" --template qa

# 添加Prompt
python run.py prompt add --app 我的问答助手 --name "问答prompt" --content "你是一个专业的客服，请回答用户问题"

# 运行测试
python run.py evaluate --app 我的问答助手

# 启动服务
python run.py serve --app 我的问答助手 --port 8080
```

### Docker部署

```bash
# 构建镜像
docker build -t langops-enterprise:latest .

# 运行容器
docker run -d -p 8080:8080 -v ./data:/app/data langops-enterprise:latest
```

## 十、核心API

### 版本管理

```python
from core.version import VersionManager

vm = VersionManager()
version_id = vm.create_version(app_id, {"prompt": "...", "rag_config": {...}})
versions = vm.list_versions(app_id)
vm.rollback(app_id, version_id)
```

### 效果评测

```python
from core.evaluator import Evaluator

evaluator = Evaluator()
results = evaluator.evaluate(app_id, test_cases, dimensions=["accuracy", "relevance", "compliance"])
```

### A/B测试

```python
from core.abtest import ABTestEngine

engine = ABTestEngine()
test_id = engine.create_test(app_id, {"version_a": {...}, "version_b": {...}})
```
